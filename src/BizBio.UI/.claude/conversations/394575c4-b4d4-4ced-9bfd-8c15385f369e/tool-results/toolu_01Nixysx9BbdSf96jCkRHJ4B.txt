    54→## Key Concepts
    55→
    56→### Options vs Extras
    57→
    58→**Options** (Required Choices):
    59→- `MinRequired >= 1`
    60→- Examples: "Choose a drink", "Select size", "Pick a side"
    61→- Customer MUST select one
    62→
    63→**Extras** (Optional Add-ons):
    64→- `MinRequired = 0`
    65→- Examples: "Add extra cheese", "Would you like more?"
    66→- Customer CAN select, but doesn't have to
    67→
    68→### The Index Pattern
    69→
    70→Instead of duplicating data:
    71→```json
    72→// ❌ Traditional
    73→{ "options": [{ ...full data... }] }  // Repeated in every variant
    74→```
    75→
    76→MRD uses indices:
    77→```json
    78→// ✅ MRD Pattern
    79→{ "option_indices": [0, 1, 2] }  // Just references menu.options[]
    80→```

   103→## How It Works
   104→
   105→```
   106→┌─────────────────────────────────────────────────────┐
   107→│                    Frontend                         │
   108→│  - Loads menu once                                  │
   109→│  - Direct array access: menu.options[0]             │
   110→│  - Simple cart: { variantId, selectedOptions[] }    │
   111→└──────────────────┬──────────────────────────────────┘
   112→                   │
   113→                   │ GET /api/v2/menu/123
   114→                   │
   115→┌──────────────────▼──────────────────────────────────┐
   116→│              MRDStyleMenuController                 │
   117→│  - Receives request                                 │
   118→│  - Calls MenuMappingService                         │
   119→│  - Returns MRD-style DTO                            │
   120→└──────────────────┬──────────────────────────────────┘
   121→                   │
   122→                   │ MapCatalogToMenu(catalog)
   123→                   │
   124→┌──────────────────▼──────────────────────────────────┐
   125→│            MenuMappingService                       │
   126→│  1. Build global options[] array                    │
   127→│  2. Build global extras[] array                     │
   128→│  3. Create index maps: GroupId → ArrayIndex        │
   129→│  4. Map variants with option_indices/extra_indices  │
   130→└──────────────────┬──────────────────────────────────┘
   131→                   │
   132→                   │ EF Core query
   133→                   │
   134→┌──────────────────▼──────────────────────────────────┐
   135→│               Database (Normalized)                 │
   136→│  CatalogItem                                        │
   137→│    ├── Variants[]                                   │
   138→│    └── ExtraGroupLinks[]                            │
   139→│          └── ExtraGroup                             │
   140→│                ├── MinRequired (0=Extra, ≥1=Option) │
   141→│                └── GroupItems[] → Extra             │
   142→└─────────────────────────────────────────────────────┘
   143→```

   319→## Common Questions
   320→
   321→### Q: Do I need to change my database?
   322→**A:** No! Your schema is already perfect. The mapping happens at the API layer.
   323→
   324→### Q: What if I add new option groups?
   325→**A:** Just create them in the database. The mapping service automatically picks them up and adds them to the options/extras arrays.
   326→
   327→### Q: How do I handle variant-specific options?
   328→**A:** Use the `VariantId` field in `CatalogItemExtraGroupLink`:
   329→```csharp
   330→// Link applies to specific variant
   331→new CatalogItemExtraGroupLink {
   332→    CatalogItemId = 1,
   333→    ExtraGroupId = 10,
   334→    VariantId = 123  // Only for this variant
   335→}
   336→
   337→// Link applies to all variants
   338→new CatalogItemExtraGroupLink {
   339→    CatalogItemId = 1,
   340→    ExtraGroupId = 10,
   341→    VariantId = null  // For all variants
   342→}
   343→```
   344→
   345→### Q: Can I still use the old API format?
   346→**A:** Yes! Keep your existing MenuController. The new MRDStyleMenuController runs in parallel.
