     1→# BizBio Multi-Product Subscription Architecture - Implementation Plan
     2→
     3→## Overview
     4→Transform BizBio from single-subscription to multi-product subscription model with unified entity management.
     5→
     6→## Core Architectural Principles
     7→
     8→### 1. Products
     9→Three distinct subscription products:
    10→- **Cards** - Digital business card profiles
    11→- **Menu** - Restaurant/venue menu management
    12→- **Catalog** - Product catalog for stores/businesses
    13→
    14→Each product has own subscription tier with independent trial periods, billed together monthly.
    15→
    16→### 2. Entity Unification
    17→Single `Entities` table replaces separate Restaurant/Company/Profile tables:
    18→- Entity represents the business/organization/venue
    19→- EntityType enum distinguishes Restaurant/Store/Venue/Organization
    20→- Each entity can have multiple catalogs/menus (per tier limits)
    21→
    22→### 3. Library/Menu/Catalog Terminology
    23→**Important**: Library, Menu, and Catalog are the **same thing** - just different UI terminology:
    24→- "Library" = user's private items (IsActive=false or no catalog assignment)
    25→- "Menu" = published catalog for restaurants (IsActive=true)
    26→- "Catalog" = published catalog for stores (IsActive=true)
    27→
    28→All stored in single `Catalogs` table. No CatalogType needed.
    29→
    30→### 4. Price Override Pattern
    31→Menu items can be shared across catalogs with price overrides:
    32→- CatalogItem has `ParentCatalogItemId` (nullable)
    33→- If null = master/template item
    34→- If set = reference to master with optional price override
    35→- `PriceOverride` (nullable decimal) - null means use parent price
    36→

    43→#### Entities Table
    44→Replaces separate Restaurant/Company/Profile tables.
    45→
    46→```sql
    47→CREATE TABLE Entities (
    48→    Id INT PRIMARY KEY IDENTITY,
    49→    UserId INT NOT NULL FOREIGN KEY REFERENCES Users(Id),
    50→    EntityType INT NOT NULL,  -- 0=Restaurant, 1=Store, 2=Venue, 3=Organization
    51→    Name NVARCHAR(255) NOT NULL,
    52→    Slug NVARCHAR(255) NOT NULL UNIQUE,
    53→    Description NVARCHAR(2000),
    54→    Logo NVARCHAR(500),
    55→
    56→    -- Contact Info
    57→    Address NVARCHAR(255),
    58→    City NVARCHAR(100),
    59→    PostalCode NVARCHAR(20),
    60→    Phone NVARCHAR(20),
    61→    Email NVARCHAR(255),
    62→    Website NVARCHAR(500),
    63→
    64→    -- Settings
    65→    Currency NVARCHAR(3) DEFAULT 'ZAR',
    66→    Timezone NVARCHAR(50) DEFAULT 'Africa/Johannesburg',
    67→
    68→    -- Metadata
    69→    SortOrder INT DEFAULT 0,
    70→    IsActive BIT DEFAULT 1,
    71→    CreatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
    72→    UpdatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),

    82→#### ProductSubscriptions Table
    83→Replaces single Subscriptions table with per-product subscriptions.
    84→
    85→```sql
    86→CREATE TABLE ProductSubscriptions (
    87→    Id INT PRIMARY KEY IDENTITY,
    88→    UserId INT NOT NULL FOREIGN KEY REFERENCES Users(Id),
    89→    ProductType INT NOT NULL,  -- 0=Cards, 1=Menu, 2=Catalog
    90→    TierId INT NOT NULL FOREIGN KEY REFERENCES SubscriptionTiers(Id),
    91→
    92→    -- Trial tracking per product
    93→    IsTrialActive BIT DEFAULT 1,
    94→    TrialStartDate DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
    95→    TrialEndDate DATETIME2 NOT NULL,
    96→    TrialDaysRemaining AS DATEDIFF(DAY, GETUTCDATE(), TrialEndDate),
    97→
    98→    -- Billing
    99→    Status INT NOT NULL DEFAULT 0,  -- 0=Trial, 1=Active, 2=Cancelled, 3=Expired
   100→    BillingCycle INT DEFAULT 0,  -- 0=Monthly, 1=Yearly
   101→    CurrentPeriodStart DATETIME2,
   102→    CurrentPeriodEnd DATETIME2,
   103→    NextBillingDate DATETIME2,
   104→
   105→    -- Metadata
   106→    IsActive BIT DEFAULT 1,
   107→    CreatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
   108→    UpdatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
   109→    CancelledAt DATETIME2,
   110→
   111→    UNIQUE (UserId, ProductType),  -- One subscription per product per user
   112→    INDEX IX_ProductSubscriptions_UserId (UserId),
   113→    INDEX IX_ProductSubscriptions_Status (Status)
   114→);

   168→public decimal EffectivePrice
   169→{
   170→    get
   171→    {
   172→        if (PriceOverride.HasValue) return PriceOverride.Value;
   173→        if (ParentCatalogItemId.HasValue && ParentCatalogItem != null)
   174→            return ParentCatalogItem.EffectivePrice;
   175→        return Price;

   180→#### Categories Table
   181→Categories are now entity-level (each entity has its own categories).
   182→
   183→```sql
   184→-- Add EntityId to Categories table
   185→ALTER TABLE Categories ADD EntityId INT NULL
   186→    FOREIGN KEY REFERENCES Entities(Id) ON DELETE CASCADE;
   187→
   188→-- Migration: Link existing categories to their entity via catalog
   189→UPDATE c
   190→SET c.EntityId = cat.EntityId
   191→FROM Categories c
   192→INNER JOIN CatalogCategories cc ON cc.CategoryId = c.Id
   193→INNER JOIN Catalogs cat ON cat.Id = cc.CatalogId;
   194→
   195→-- Make EntityId required
   196→ALTER TABLE Categories ALTER COLUMN EntityId INT NOT NULL;
   197→
   198→-- Remove UserId (categories now owned by entity, not user)
   199→ALTER TABLE Categories DROP COLUMN UserId;
   200→
   201→CREATE INDEX IX_Categories_EntityId ON Categories(EntityId);
   202→```

   219→```

   256→## URL Structure
   257→
   258→### Frontend (Menu Viewing)
   259→
   260→#### Single Catalog Per Entity
   261→```
   262→/{entity_slug}
   263→Example: /the-cocktail-lounge
   264→```
   265→
   266→#### Multiple Catalogs Per Entity
   267→**Both formats must be supported:**
   268→
   269→Format 1 (Underscore):
   270→```
   271→/{entity_slug}_{catalog_slug}
   272→Example: /the-cocktail-lounge_evening-menu
   273→```
   274→
   275→Format 2 (Slash):
   276→```
   277→/{entity_slug}/{catalog_slug}
   278→Example: /the-cocktail-lounge/evening-menu
   279→```
   280→
   281→**Frontend Routing Logic:**
   282→```csharp
   283→// Try underscore format first: entity_catalog
   284→var parts = slug.Split('_');
   285→if (parts.Length == 2)
   286→{
   287→    var catalog = await _context.Catalogs
   288→        .Include(c => c.Entity)
   289→        .FirstOrDefaultAsync(c => c.Entity.Slug == parts[0] && c.Slug == parts[1] && c.IsActive);
   290→    if (catalog != null) return catalog;
   291→}
   292→
   293→// Try slash format: entity/catalog
   294→if (!string.IsNullOrEmpty(catalogSlug))
   295→{
   296→    var catalog = await _context.Catalogs
   297→        .Include(c => c.Entity)
   298→        .FirstOrDefaultAsync(c => c.Entity.Slug == entitySlug && c.Slug == catalogSlug && c.IsActive);
   299→    if (catalog != null) return catalog;
   300→}
   301→
   302→// No catalog slug, get default (first active catalog for entity)
   303→return await _context.Catalogs
   304→    .Where(c => c.Entity.Slug == entitySlug && c.IsActive)
   305→    .OrderBy(c => c.SortOrder)
   306→    .FirstOrDefaultAsync();
   307→```
   308→
   309→### API Endpoints
   310→API uses slash format exclusively:
   311→```
   312→GET /api/v1/entities/{entityId}/catalogs/{catalogId}
   313→PUT /api/v1/entities/{entityId}/catalogs/{catalogId}
   314→```

   397→      → Option appears in option group

   412→## Library Item Sharing Pattern
   413→
   414→### Scenario: Restaurant with Multiple Menus
   415→```
   416→Entity: "The Cocktail Lounge"
   417→├─ Catalog: "Evening Menu"
   418→│  ├─ Mojito (Price: R95) [master item]
   419→│  └─ Caesar Salad (Price: R85) [master item]
   420→│
   421→└─ Catalog: "Happy Hour Menu"
   422→   ├─ Mojito (Price Override: R65) [references master, different price]
   423→   └─ Caesar Salad (Price: R85) [references master, same price]
   424→```
   425→
   426→**Database Representation:**
   427→```sql
   428→-- Master items (ParentCatalogItemId = NULL)
   429→INSERT INTO CatalogItems (Name, Price, CatalogId, ParentCatalogItemId)
   430→VALUES ('Mojito', 95.00, 1, NULL);  -- ID: 100
   431→
   432→INSERT INTO CatalogItems (Name, Price, CatalogId, ParentCatalogItemId)
   433→VALUES ('Caesar Salad', 85.00, 1, NULL);  -- ID: 101
   434→
   435→-- Happy Hour references with override
   436→INSERT INTO CatalogItems (Name, Price, CatalogId, ParentCatalogItemId, PriceOverride)
   437→VALUES ('Mojito', 95.00, 2, 100, 65.00);  -- Override price to R65
   438→
   439→INSERT INTO CatalogItems (Name, Price, CatalogId, ParentCatalogItemId, PriceOverride)
   440→VALUES ('Caesar Salad', 85.00, 2, 101, NULL);  -- No override, use parent price
   441→```
   442→
   443→### Cross-Entity Sharing
   444→**Decision**: Item sharing is limited to same entity only.
   445→
   446→When user wants to use an item from Entity A in Entity B:
   447→- **Copy only** - Creates new master item in Entity B catalog
   448→- No ParentCatalogItemId link across entities
   449→- Changes to Entity A item don't affect Entity B
   450→- User can manually sync changes if needed
   451→
   452→**Rationale**: Keeps data ownership clear, prevents confusion when different entities (e.g., different restaurants) want different pricing/descriptions for similar items.

   456→## Subscription Billing Logic
   457→
   458→### Trial Periods
   459→Each product has independent 7-day trial:
   460→```
   461→User signs up on Jan 1:
   462→- Menu product trial: Jan 1-7
   463→- User adds Catalog product on Jan 3
   464→- Catalog product trial: Jan 3-9
   465→
   466→All trials must end before billing starts
   467→First billing date: Jan 10 (after both trials end)
   468→```
   469→
   470→### Pro-Rata Billing
   471→When adding product mid-billing cycle:
   472→```
   473→Billing cycle: Jan 1-31
   474→Menu subscription: R299/month
   475→
   476→User adds Catalog on Jan 15 (after trial ends):
   477→- Catalog full price: R199/month
   478→- Days remaining in cycle: 16 days
   479→- Pro-rata amount: R199 × (16/31) = R102.71
   480→
   481→Next bill (Jan 31):
   482→- Menu: R299
   483→- Catalog (pro-rata): R102.71
   484→- Total: R401.71
   485→
   486→Feb 1 onwards:
   487→- Menu: R299
   488→- Catalog: R199
   489→- Total: R498/month
   490→```
   491→
   492→### Combined Billing
   493→All active product subscriptions billed together on same date:
   494→```
   495→Invoice #123 - Jan 31, 2025
   496→─────────────────────────────
   497→Menu - Professional Plan       R299.00
   498→Catalog - Business Plan        R199.00
   499→Cards - Pro Plan               R149.00
   500→                          ─────────────
   501→Subtotal                       R647.00
   502→VAT (15%)                       R97.05
   503→                          ─────────────
   504→Total Due                      R744.05
   505→```

   982→### Phase 6: Migrate Categories to Entity-Level
   983→```sql
   984→-- Add EntityId to Categories (nullable first)
   985→ALTER TABLE Categories ADD EntityId INT NULL;
   986→
   987→-- Link categories to entities via their catalogs
   988→UPDATE c
   989→SET c.EntityId = cat.EntityId
   990→FROM Categories c
   991→INNER JOIN CatalogCategories cc ON cc.CategoryId = c.Id
   992→INNER JOIN Catalogs cat ON cat.Id = cc.CatalogId;
   993→
   994→-- For categories not yet in CatalogCategories, link via old CatalogId if exists
   995→UPDATE c
   996→SET c.EntityId = cat.EntityId
   997→FROM Categories c
   998→INNER JOIN Catalogs cat ON cat.Id = c.CatalogId
   999→WHERE c.EntityId IS NULL AND c.CatalogId IS NOT NULL;
  1000→
  1001→-- Make EntityId required
  1002→ALTER TABLE Categories ALTER COLUMN EntityId INT NOT NULL;
  1003→
  1004→-- Add foreign key and index
  1005→ALTER TABLE Categories ADD CONSTRAINT FK_Categories_Entities
  1006→    FOREIGN KEY (EntityId) REFERENCES Entities(Id) ON DELETE CASCADE;
  1007→CREATE INDEX IX_Categories_EntityId ON Categories(EntityId);
  1008→
  1009→-- Remove UserId (categories now owned by entity, not user)
  1010→ALTER TABLE Categories DROP COLUMN UserId;
  1011→
  1012→-- Create CatalogCategories join table entries if not already done
  1013→INSERT INTO CatalogCategories (CatalogId, CategoryId, SortOrder, CreatedAt)
  1014→SELECT DISTINCT
  1015→    c.CatalogId,
  1016→    cat.Id as CategoryId,
  1017→    cat.SortOrder,
  1018→    GETUTCDATE()
  1019→FROM Categories cat
  1020→INNER JOIN Catalogs c ON c.EntityId = cat.EntityId
  1021→WHERE NOT EXISTS (
  1022→    SELECT 1 FROM CatalogCategories cc
  1023→    WHERE cc.CatalogId = c.Id AND cc.CategoryId = cat.Id
  1024→);
  1025→
  1026→-- Remove old CatalogId from Categories if it exists
  1027→IF EXISTS (SELECT * FROM sys.columns WHERE object_id = OBJECT_ID('Categories') AND name = 'CatalogId')
  1028→BEGIN
  1029→    ALTER TABLE Categories DROP COLUMN CatalogId;
  1030→END

  1146→## Decisions Made
  1147→
  1148→1. **Cross-entity item sharing**: Same entity only - ParentCatalogItemId references only work within same entity
  1149→
  1150→2. **Entity limit enforcement**: Free tier allows 1 entity (no forced upgrade)
  1151→
  1152→3. **Category sharing scope**: Entity-level - each entity has its own categories (Categories table has EntityId FK)
  1153→
  1154→4. **URL format**:
  1155→   - **Frontend viewing**: Support both `/{entity_slug}_{catalog_slug}` AND `/{entity_slug}/{catalog_slug}` for multiple catalogs
  1156→   - **Single catalog**: `/{entity_slug}` only
  1157→   - **API**: `/{entity}/{catalog}` format exclusively
  1158→
  1159→5. **Bundle assignment**: Catalog-level - bundles belong to specific catalog (not shared across entity's catalogs)
